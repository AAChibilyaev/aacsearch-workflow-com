# AACSearch OS — Документация для клиентов (Customer Guide)

> **Полное руководство для клиентов AACSearch OS.** Все разделы админки, все страницы,
> все возможности — от регистрации до продвинутого поиска.

---

# 1. НАЧАЛО РАБОТЫ

## 1.1 Регистрация

```
1. Перейдите на https://aacsearch.ru
2. Нажмите "Начать бесплатно" или "Sign Up"
3. Введите email и пароль
4. Подтвердите email (письмо с ссылкой)
5. Автоматически создаётся ваш workspace (тенант)
6. Вы попадаете в Dashboard
```

## 1.2 Dashboard (`/admin`)

Главная страница после входа. Показывает:
- Краткую сводку: количество документов, коллекций, поисковых запросов
- Быстрые действия: создать коллекцию, добавить документы, получить API ключ
- Последние новости и обновления платформы

# 2. ДОКУМЕНТЫ И КОЛЛЕКЦИИ

## 2.1 Коллекции (`/admin/collections/collection-definitions`)

Коллекция = схема данных для поиска. Например: "Товары", "Статьи", "FAQ".

### Создание коллекции:

1. Нажмите "Create New" → "Collection Definition"
2. Заполните поля:
   - **Name** — название (на русском, английском, немецком)
   - **Slug** — техническое имя (латиницей, например `products`)
   - **Fields** — поля коллекции
3. Для каждого поля укажите:
   - **Name** — имя поля (`title`, `price`, `brand`)
   - **Field Type** — тип (`text`, `number`, `checkbox`, `date`, ...)
   - **Searchable** — можно ли искать по этому полю
   - **Facetable** — можно ли фильтровать
   - **Sortable** — можно ли сортировать
4. Сохраните

### Типы полей:

| Тип | Пример | Поиск | Фильтр | Сортировка |
|-----|--------|:---:|:---:|:---:|
| Text | Название товара | ✅ | ✅ | ❌ |
| Number | Цена, рейтинг | ❌ | ✅ | ✅ |
| Checkbox | В наличии | ❌ | ✅ | ❌ |
| Date | Дата публикации | ❌ | ✅ | ✅ |
| Select | Категория | ❌ | ✅ | ❌ |

## 2.2 Документы (`/admin/collections/documents`)

Документ = одна запись в коллекции. Например: товар, статья, FAQ-вопрос.

### Добавление документов:

**Способ 1: Вручную через админку**
1. Перейдите в "Documents"
2. Нажмите "Create New"
3. Выберите коллекцию (Definition)
4. Заполните данные в JSON-формате
5. Сохраните → документ автоматически индексируется

**Способ 2: Через API**
```bash
curl -X POST "https://search.aacsearch.ru/api/documents" \
  -H "Authorization: api-keys API-Key YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Ноутбук Dell XPS",
    "definition": "ID_коллекции",
    "data": {"title": "Dell XPS 13", "price": 129990, "brand": "Dell"},
    "tenant": 123
  }'
```

**Способ 3: Массовый импорт**
1. Перейдите в "Documents" → "Import"
2. Загрузите JSONL-файл (один JSON-объект на строку)
3. Выберите коллекцию
4. Нажмите "Import"

**Способ 4: Через интеграции (Nango)**
1. Подключите Shopify/WordPress/Airtable/...
2. Данные синхронизируются автоматически

# 3. НАСТРОЙКА ПОИСКА

## 3.1 Tenant Settings (`/admin/tenant-settings`)

Здесь настраивается поведение поиска для вашего workspace.

### Поисковые поля (Searchable Fields)
Выберите, по каким полям искать и с каким весом:
- `title` (вес: 4) — самое важное
- `description` (вес: 2)
- `brand` (вес: 3)

### Фильтры (Facet Fields)
Выберите поля для фильтрации в поисковом интерфейсе:
- `brand`, `category`, `price`

### Опечатки (Typo Tolerance)
- Количество допустимых опечаток: 0-2
- Минимальная длина слова для опечаток

### Ранжирование (Ranking)
- Поле сортировки по умолчанию
- Приоритет точных совпадений

## 3.2 Синонимы (Synonyms)

Настройте синонимы, чтобы поиск "ноутбук" находил "лэптоп":
```
ноутбук → лэптоп, ноут, портативный компьютер
смартфон → телефон, мобильный
купить → приобрести, заказать
```

## 3.3 Курация (Curation)

Закрепите важные товары в результатах поиска:
- Поиск "распродажа" → сначала показывать акционные товары
- Поиск "новинки" → сначала новые поступления
- Скрыть товары, которых нет в наличии

## 3.4 Семантический поиск (AI)

Включите поиск по смыслу:
- "удобный ноутбук для работы" найдёт бизнес-ноутбуки
- "бюджетный смартфон" найдёт телефоны до 20000
- Работает на русском, английском и немецком

# 4. ИНТЕГРАЦИИ

## 4.1 Каталог интеграций (`/admin/integrations`)

Подключите внешние источники данных. Доступно 189+ сервисов.

### Популярные интеграции:

**Интернет-магазины:**
- Shopify — товары, заказы, клиенты
- WooCommerce — товары WordPress-магазина
- BigCommerce — корпоративный e-commerce

**CRM:**
- Salesforce — контакты, сделки, компании
- HubSpot — клиентская база
- Pipedrive — воронка продаж

**Контент:**
- WordPress — посты, страницы, медиа
- Contentful — headless CMS
- Google Drive — документы, таблицы

**Базы данных:**
- Airtable — таблицы и базы
- Notion — рабочие пространства
- Google Sheets — электронные таблицы

### Как подключить:

1. Перейдите в "Integrations"
2. Выберите провайдера из каталога
3. Нажмите "Connect"
4. Авторизуйтесь в сервисе (OAuth)
5. Данные начнут синхронизироваться автоматически
6. Отслеживайте статус в "Connections"

# 5. ПОИСК НА САЙТЕ

## 5.1 Виджет (`/admin/widget`)

Получите готовый код для вставки поиска на ваш сайт.

### Вариант 1: CDN Widget (1 строка)
```html
<script src="https://cdn.aacsearch.ru/widget.js"></script>
<script>
  AACSearch.init({
    apiKey: 'YOUR_SCOPED_KEY',
    collection: 'products',
    container: '#search'
  });
</script>
```

### Вариант 2: React компонент
```bash
npm install @aacsearch/ui
```
```tsx
import { AACSearchProvider, SearchBox, Hits, Facets } from '@aacsearch/ui';
<AACSearchProvider apiKey="..." collection="products">
  <SearchBox placeholder="Поиск товаров..." />
  <Facets facets={['brand', 'price']} />
  <Hits />
</AACSearchProvider>
```

### Вариант 3: SDK (полный контроль)
```bash
npm install @aacsearch/sdk
```
```ts
import { AACSearch } from '@aacsearch/sdk';
const client = new AACSearch({ apiKey: 'YOUR_KEY' });
const results = await client.multiSearch.perform({
  searches: [{ collection: 'products', q: 'laptop', query_by: 'title' }]
});
```

# 9. БЕЗОПАСНОСТЬ

## 9.1 API Ключи (`/admin/api-keys`)

Создавайте ключи для разных целей:
- **Search-only** — только поиск (для фронтенда)
- **Read-only** — чтение данных
- **Write** — создание и обновление документов
- **Admin** — полный доступ

### Scoped Keys (рекомендуется для фронтенда):
- Ограничены одним тенантом
- Имеют срок действия
- Не могут изменить данные
- Безопасны для использования в браузере

## 9.2 Scoped Key для виджета

```bash
curl -X POST "https://search.aacsearch.ru/api/v1/keys/scoped" \
  -H "Authorization: api-keys API-Key YOUR_ADMIN_KEY" \
  -d '{"tenant":123,"ttl_seconds":3600}'
# → {"scopedKey":"...","expiresAt":1712345678}
```
